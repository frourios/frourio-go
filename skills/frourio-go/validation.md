# Validation

frourio-go uses [`github.com/go-playground/validator/v10`](https://pkg.go.dev/github.com/go-playground/validator/v10).
Tag any spec field with `validate:"..."` and the runtime enforces it.

## Where Validation Runs

| Stage              | When                          | Failure result |
|--------------------|-------------------------------|----------------|
| Request decode     | Before middleware/handler     | 422 Unprocessable Entity |
| Middleware context | Before next middleware/handler | 500 (server bug) |
| Response body      | Before writing the response   | 500 (server bug) |

Decode failures (e.g. `"abc"` in an `int` query) also surface as 422 with the
same error envelope.

## Tag Locations

```go
type FrourioSpec struct {
    Get struct {
        Param int `validate:"required,gte=1"`
        Query struct {
            Limit *int `json:"limit" validate:"omitempty,min=1,max=100"`
        }
        Header struct {
            XAPIKey string `json:"x-api-key" validate:"required"`
        }
        Res struct {
            Status200 struct {
                Body string `json:"body" validate:"required"`
            }
        }
    }
}
```

Tags are passed through verbatim ‚Äî anything validator/v10 supports works,
including cross-field rules (`eqfield`, `gtfield`, ‚Ä¶) and registered custom
validators.

For optional pointer fields, prefix `omitempty` so the rest of the rule only
runs when the value is present:

```go
Limit *int `json:"limit" validate:"omitempty,min=1,max=100"`
```

## Error Response Shape (422)

The runtime emits a structured JSON error:

```json
{
  "status": 422,
  "error": "Unprocessable Entity",
  "issues": [
    { "path": ["body", "name"],   "message": "required" },
    { "path": ["query", "limit"], "message": "min" }
  ]
}
```

`path` is an **array**, rooted at the request part (`"params"`, `"query"`,
`"header"`, `"body"`) followed by the field name. For path-parameter errors
the path is `["params", "<slug>"]` ‚Äî the slug name is the directory name (the
ancestor that declared the `Param`), not always the leaf. `message` is the
failing validator tag (e.g. `required`, `min`, `email`).

The `FrourioError` schema is included in the generated OpenAPI as a default
`422` response on **every** operation; clients generated by oapi-codegen get a
typed binding for it.

Pure decode errors (e.g., `?limit=abc` failing `strconv.Atoi`) also surface
as 422 with a single issue describing the part that failed to decode.

500 responses (handler returned `nil`, response validation failed,
middleware error, etc.) reuse the same envelope without an `issues` array.

## Response Validation

Every declared `Body` is validated before writing. Use this to catch
server-side bugs early:

```go
Res struct {
    Status200 struct {
        Body string `json:"body" validate:"required"`
    }
}
```

If the handler returns `Body: ""`, the response is logged and written as 500.
This is intentional: the client did nothing wrong, but the server promised a
non-empty body.

For test apps that check 500 paths, look for response-validation triggers
rather than client misuse.

## Middleware Context Validation

Each `Middleware.All.Context` and per-method `Middleware.<Method>.Context` is
validated after the middleware sets it. Treat required fields as a contract:

```go
Middleware struct {
    All struct {
        Context struct {
            UserID  string `json:"userId" validate:"required"`
            TraceID string `json:"traceId" validate:"required"`
        }
    }
}
```

If the middleware forgets to set `UserID`, the request fails with 500 before
the handler runs. See [middleware.md](middleware.md).

## Common Tags

Useful day-to-day tags:

- `required` ‚Äî non-zero value
- `omitempty` ‚Äî skip remaining rules if zero
- `min=`, `max=`, `len=` ‚Äî for strings, numbers, slices, maps
- `gte=`, `lte=`, `gt=`, `lt=` ‚Äî numeric comparisons
- `email`, `url`, `uuid`
- `oneof=foo bar baz` ‚Äî allow listed values
- `dive` ‚Äî apply rules to slice elements

## OpenAPI Visibility

`validate` tags are **not** standard OpenAPI. The generator preserves the
verbatim tag string as an `x-go-validate` extension on the schema. It does
not convert tags into native OpenAPI keywords like `minimum` / `maximum` ‚Äî
clients that want to enforce the same rules need to read the extension. See
[openapi.md](openapi.md).
